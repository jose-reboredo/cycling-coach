# Post-mortem — v9.6.1 OAuth callback CSP regression

## Timeline (UTC, 2026-04-30)

- ~12:00 — v9.5.1 deployed (Sprint 3 Phase 2 — `#15` strict CSP `script-src 'self'`). Smoke focused on header presence and `/api/*` method allowlist; OAuth flow not exercised.
- ~13:00 — v9.6.0 deployed (Sprint 4 Phase 1 — clubs 4-tab shell). CSP unchanged from v9.5.1.
- ~14:30 — First user report: "no se autentifica con strava, el dashboard se queda cargando, loading".
- ~15:00 — Root cause identified via DevTools console + Worker logs: strict CSP blocked the `/callback` inline `<script>` that writes Strava tokens to `localStorage` and redirects to `/dashboard`. Browser silently dropped the script — no `pageerror`, no JS exception, just a stalled page.
- ~15:30 — v9.6.1 hotfix shipped (commit `22636c7`).

## User-visible symptom

Users completing Strava OAuth land on `/callback` showing "Loading dashboard…" that never advances. Tokens never written, redirect never fires.

## Root cause (`src/worker.js:~2900` callback page)

Sprint 3 `#15` introduced `script-src 'self'` in the `Content-Security-Policy` header applied to every Worker response via `withSecurityHeaders`. The `/callback` handler returns an HTML page containing an inline `<script>` tag that:

1. Reads the OAuth `code` + `state` from the URL
2. Writes tokens to `localStorage`
3. Redirects to `/dashboard` (or `/dashboard/today` on PWA)

Strict CSP `script-src 'self'` disallows inline `<script>` execution unless a nonce is present. The browser silently dropped the script. The legacy-parity audit at `#15` merge time looked at visual chrome but did not catalogue inline-script surfaces.

## Fix

Per-request nonce CSP for `/callback` only — commit `22636c7`:

- New helper `cspWithScriptNonce(nonce)` rewrites the `script-src` directive to `'self' 'nonce-{value}'`, leaving every other directive untouched.
- `/callback` handler generates `crypto.randomUUID().replace(/-/g, '')` per request, passes it to `callbackPage()`, and sets the response's `Content-Security-Policy` to `cspWithScriptNonce(nonce)`.
- `callbackPage()` gained a `nonce` param; both inline `<script>` tags render as `<script nonce="{value}">`.
- Strict CSP preserved on every other route — the per-response override only loosens `script-src` for `/callback`.

## Prevention rule

**Legacy-parity audit must include CSP / inline-script surfaces, not just visual chrome.** When merging any change to security headers, list every route that returns inline-script HTML and verify the header is compatible with each. Promoted to `0-learnings.md` Rule #1 (refined from Sprint 1 Improvement #1).
