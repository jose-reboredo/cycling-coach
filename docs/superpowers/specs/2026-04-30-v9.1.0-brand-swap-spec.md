# v9.1.0 — Brand Swap Spec

**Goal:** Rename "Cycling Coach" → "Cadence Club", swap accent palette to Brass `#B8956A` + Forest Green `#2C5530`, update copy voice per the v2.0 redesign brief Section 9.2, and fix the worst mobile-first offenders. **No new pages. No auth changes. No D1 migrations.**

**Why this spec exists:** post-deploy of v9.0.0 (F4 invite-by-link), Jose locked v9.1.0 = brand swap only. This spec captures the file inventory, color migration rules, and audit findings so tomorrow's implementation is mechanical, not exploratory.

**Out of scope:** new pages (homepage 3-col, onboarding flow, settings, B2B placeholder — all v9.2.0). Email/password auth (excluded entirely; Strava OAuth remains the only auth path). Anthropic API key client-side migration.

### 1.2 `tone=` semantic remap

The codebase has ~17 sites passing `tone="accent"` or `tone="success"` to `<Eyebrow>` and `<Pill>`. Today both render in molten orange + green generically. Post-v9.1.0 the components themselves don't change, but the *rendering* changes via the new tokens, AND we need to triage which sites are semantically wrong under the new "brass = active, green = completed" rule.

**Rule:** Brass = "now / next / active / interactive." Forest Green = "done / earned / past." Anything ambiguous → **neutral gray**, not green.

## §4 Mobile-first audit findings

Components without ANY `@media` query — surveyed via grep. Triaged into **fix-in-v9.1.0** vs **defer-to-v9.2.0**.

### Fix in v9.1.0 (4 components)

These are **page-level or large-surface** components where desktop-first CSS materially harms mobile UX:

1. **`OnboardingModal/OnboardingModal.module.css`** — fixed widths (e.g., `max-width: 520px`), three-column form sections that don't reflow. On 375px iPhone the modal overflows the viewport horizontally. Fix: `max-width: min(520px, calc(100vw - 32px))` + stack form columns at <600px.

2. **`GoalEventCard/GoalEventCard.module.css`** — large card with date picker + multi-column meta. Probably wraps awkwardly on mobile. Audit + add a `@media (max-width: 640px)` breakpoint for column stacking.

3. **`RideFeedback/RideFeedback.module.css`** — spans the dashboard width with fixed-width inner layout. Likely overflow on small screens. Stack the verdict + body sections vertically <768px.

4. **`WinsTimeline/WinsTimeline.module.css`** — horizontal timeline that almost certainly clips on mobile. Either rotate to vertical-stack at <640px OR make the horizontal container `overflow-x: auto` with snap points.

### Defer to v9.2.0 (9 components)

These are **atoms or small components** where the absence of media queries is fine — they're flexible by default or will be re-styled when v9.2.0's mobile-first new pages land:

- `Button` — atom, sized by content
- `Eyebrow`, `Pill`, `ZonePill` — inline atoms
- `GrainOverlay` — fixed full-bleed overlay, viewport-relative
- `ProgressRing` — sized by SVG props
- `UserMenu` — popover, sized by content
- `WhatsNew` — small popup
- `ClubCreateModal` — already audited, uses parent `OnboardingModal` pattern; will inherit fix from #1 above

**Audit-only items** (verify but don't fix unless broken):
- `Card`, `Container` — generic primitives, expected to be flexible
- `BikeMark`, `BottomNav` — already mobile-aware (BottomNav exists *for* mobile)

## §5 Implementation order

Do these in order — each step is independently safe to commit + push:

1. **Tokens swap** (`design/tokens.css`) — single file, 8 lines changed. Adds `--c-success-deep`. Verify `--c-accent-light` AA contrast in DevTools before commit. Smoke: every existing component should render with the new palette automatically.
2. **Tone= remap** (5 sites listed in §1.2) — small surgical edits.
3. **Hardcoded color audit** (§1.3) — grep first, fix anything that surfaces.
4. **Brand string rename** (§2 inventory) — bulk find-and-replace, with the `--no-rename` exclusions noted.
5. **Copy voice substitutions** (§3 table) — file-by-file pass.
6. **Mobile-first fixes** (§4 fix-list, 4 components).
7. **Version bumps** v9.0.0 → v9.1.0 in 5 places (package.json ×2, package-lock, WORKER_VERSION, APP_VERSION).
8. **CHANGELOG + README "What's new in v9.1.0"** entries.

Total: ~3-4 hours of focused work. Single commit at the end OR commit per step (recommended: commit per step for cleaner CI green-gates and easier rollback if any one step regresses).

Awkward but not broken until #32 lands:
- Invite links generated in `<InviteLinkCard />` will show `cycling-coach.josem-reboredo.workers.dev/join/...` despite the "Cadence Club" brand. Users may find this jarring. Consider a short helper line under the invite URL: "Hosted at our current workspace; bookmark cadenceclub.cc once we migrate."
- `/version` `service: "Cadence Club"` reads as inconsistent with the URL. Internal-only field, low-impact.
- `index.html <title>` and meta tags — update to "Cadence Club" regardless of URL.
